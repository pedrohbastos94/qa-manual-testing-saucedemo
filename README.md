# 🧪 QA Manual Testing Project — SauceDemo

Projeto prático de **Testes Manuais** realizado durante treinamento de **Qualidade de Software (QA)**.

Este projeto teve como objetivo aplicar na prática conceitos de testes de software, incluindo análise de requisitos, criação de cenários de teste, execução de testes e registro de bugs.

---

# 📌 Sistema testado

Sistema: SauceDemo  
https://www.saucedemo.com

O SauceDemo simula um sistema de **e-commerce**, permitindo testar fluxos completos de compra.

---

# 🎯 Objetivo do projeto

Validar as principais funcionalidades do sistema garantindo que os requisitos funcionais sejam atendidos.

---

# 🔍 Funcionalidades testadas

- Login de usuário
- Listagem de produtos
- Carrinho de compras
- Checkout
- Finalização do pedido

---

# 🧪 Atividades realizadas

Durante o projeto foram realizadas as seguintes atividades de QA:

- Criação de **10 cenários de teste**
- Elaboração de **casos de teste (Step e BDD / Gherkin)**
- Execução dos testes
- Registro de **evidências**
- Identificação e documentação de **bugs**
- Elaboração de **relatório de testes**
- Análise de **riscos**
- Sugestões de **automação**

---

# 📊 Resultados dos testes

- Casos de teste executados: **10**
- Testes aprovados: **10**
- Bugs encontrados: **2**

---

# 🐞 Bugs identificados

Exemplos de problemas encontrados durante os testes:

**Bug 1 — Mensagem de erro pouco clara no login**

O sistema apresenta uma mensagem genérica ao inserir credenciais inválidas.

**Bug 2 — Feedback visual limitado ao adicionar produto**

Ao adicionar um produto ao carrinho não existe uma confirmação visual clara da ação.

- Problemas no carrinho podem impedir a finalização da compra.
- Erros no checkout podem gerar pedidos incorretos.
